GitHub Pages 部署教程

学习如何将你的静态网站免费部署到 GitHub Pages

快速导航： 返回首页 | 快速开始 | 文件夹对比

什么是 GitHub Pages？

GitHub Pages 是 GitHub 提供的一项免费静态网站托管服务。你可以直接从 GitHub 仓库托管你的个人、组织或项目网站。

完全免费

无需支付任何费用

免费域名

username.github.io

版本控制

基于 Git 管理

快速部署

推送即发布

快速开始

1

创建 GitHub 仓库

  1. 登录 GitHub 账号
  2. 点击右上角的 + 按钮，选择 New repository
  3. 填写仓库名称：
    username.github.io
    注意： 请将 username 替换为你的 GitHub 用户名，必须完全一致！
  4. 选择 Public（公开仓库）
  5. 点击 Create repository 完成创建
2

上传网站文件

将你的静态网站文件上传到仓库。必须包含一个 index.html 文件。

方式一：网页上传

  1. 在仓库页面点击 Add file → Upload files
  2. 拖拽或选择你的文件
  3. 点击 Commit changes

方式二：Git 命令行

# 克隆仓库到本地
git clone https://github.com/username/username.github.io

# 进入目录
cd username.github.io

# 添加文件
echo "# 我的网站" > index.html

# 提交并推送
git add .
git commit -m "Initial commit"
git push origin main
3

启用 GitHub Pages

  1. 进入仓库的 Settings → Pages
  2. 在 Source 部分选择 main 分支和 / (root)
  3. 点击 Save
  4. 等待 1-2 分钟，网站将自动上线

根目录 vs /docs 文件夹

GitHub Pages 支持两种部署来源，了解它们的区别，选择最适合你的方式。

根目录 (/)

  • 默认选项 - 大多数项目的首选
  • 简单直接 - 文件直接放在仓库根目录
  • 支持 Jekyll - 自动构建静态站点
  • 适合用户站点 - username.github.io
文件结构：
repository/
├── index.html
├── css/
├── js/
└── images/

/docs 文件夹

  • 分离关注点 - 网站与代码分离
  • 整洁有序 - 项目结构更清晰
  • 适合文档站 - API文档、项目文档
  • 保留构建输出 - 适合构建工具输出
文件结构：
repository/
├── docs/
│ ├── index.html
│ ├── css/
│ └── js/
├── src/
└── README.md

自定义域名（可选）

如果你有自己的域名，可以绑定到 GitHub Pages：

  1. 在 Pages 设置中输入你的域名
  2. 在你的域名服务商处添加 DNS 记录
  3. 勾选 Enforce HTTPS
# DNS 配置示例 @ 300 IN A 185.199.108.153 @ 300 IN A 185.199.109.153 www 300 IN CNAME username.github.io

常见问题

Q: 网站没有显示？

A: 检查仓库名称是否正确，确认 GitHub Pages 已启用，等待 2-5 分钟后再试。

Q: 样式不生效？

A: 检查 CSS 文件路径，确保路径大小写正确，浏览器清除缓存后刷新。

Q: 如何部署项目网站？

A: 项目网站地址是 username.github.io/repository-name，在项目 Settings → Pages 中启用即可。
